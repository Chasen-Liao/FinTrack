1. 先把定位讲清楚

答辩一开始，不要直接讲代码。先用 20 秒把系统定位说清楚： 这是一个把 股票行情、财经新闻 和 机器学习预测 结合起来的量化交易研究系统。 其中 LLM 不直接决定买卖，而是负责把新闻文本转成结构化语义特征，再交给 XGBoost 做方向预测。

你要强调的创新
LLM 做语义解析，XGBoost 做统计预测。
你要强调的约束
避免前视偏差，保证交易日对齐。
你要强调的结果
模型结果必须落到回测，而不是停留在分类分数。
代码落点 `backend/pipeline/` 代码落点 `backend/ml/` 代码落点 `backend/api/routers/`

2. 系统总体流程，老师最爱先问这个

这个问题的目标不是考你记忆力，而是看你能不能把系统说成一条清晰的数据链。 你要能从“数据从哪来”一直讲到“最后怎么显示给用户”。

高频 1老师：你这个系统的整体流程是什么？

你可以这样答： 先从 Polygon.io 拉取股票日线行情和相关新闻； 然后对新闻做 Layer 0 规则过滤，去掉空摘要、过短摘要、榜单类文章和明显无关的内容； 接着用 Layer 1 做批量 LLM 情绪分析，把新闻转成情绪、相关性、事件类别和原因解释； 之后把新闻按交易日对齐，再和 OHLC 价格数据合并做特征工程； 最后用 XGBoost 做未来 T+1 或 T+5 方向预测，并把预测概率转换成交易信号做回测。

对应实现：backend/pipeline/layer0.py、backend/pipeline/layer1.py、backend/pipeline/alignment.py、backend/ml/features.py、backend/ml/strategy_backtest.py
高频 2老师：为什么要把系统拆成这么多层？

因为不同层负责不同任务。 Layer 0 负责“降噪”，只保留与股票相关的新闻； Layer 1 负责“结构化”，把新闻文本转成模型能用的数值或类别特征； Layer 2 负责“解释”，回答用户“这条新闻为什么影响股价”； 最后机器学习模型负责“预测”。 这样拆的好处是，每一层职责单一，成本更可控，也更方便调试。

前端展示入口：frontend/src/components/PredictionPanel.tsx、frontend/src/components/RangeAnalysisPanel.tsx

3. 数据与新闻处理，这是答辩里的核心地基

老师如果开始追数据，通常会问“新闻怎么过滤”“怎么对齐交易日”“怎么知道没把未来信息泄露进去”。 这部分你一定要讲得特别具体。

3.1 新闻为什么要先过滤？

因为原始新闻噪声很高，空摘要、太短摘要、市场综述、榜单文章都会把模型带偏。先过滤能减少无效样本，提高后续 LLM 和特征工程的质量。

3.2 Layer 0 到底做了什么？

它主要做规则筛选，不做语义理解。重点是把明显不相关或信息量过低的新闻提前排掉，避免把无效文本送到更贵的 LLM 阶段。

3.3 Layer 1 为什么要批量分析？

因为单条调用成本太高。当前实现把多篇文章打包成一个 prompt 批量处理，输出 JSON 结构，便于后续数据库写入和特征聚合。

3.4 新闻怎么映射到交易日？

先把发布时间转成美国东部时间，再决定它属于当天还是下一个交易日。收盘后发布的新闻要顺延到下一交易日，避免把“收盘后才知道的信息”误算成当天可知。

你可以直接背的时间对齐回答

“我把新闻发布时间先转换成美东时间，再判断是否在 16:00 之后。若在收盘后发布，就顺延到下一个交易日；如果当天不是交易日，再继续往后找最近的可交易日。这样可以确保模型只使用当时理论上已经能获得的信息。”

对应实现：backend/pipeline/alignment.py，重点看 _resolve_trade_date_from_published()

4. 特征工程与标签，老师大概率会深挖

这部分是最容易被问细节的地方。你要能说出“有哪些特征”“为什么这么做”“怎么避免前视偏差”。

高频 3老师：你的特征都有哪些？

可以分成四类回答： 第一类是新闻基础特征，比如当日新闻数、相关新闻数、正负中性数量、情绪分数和情绪强度； 第二类是滚动统计特征，比如 3 日、5 日、10 日的情绪均值、新闻总量和情绪动量； 第三类是事件类别特征，比如 earnings、product、regulatory、macro、analyst、management、industry 等； 第四类是价格技术特征，比如历史收益率、波动率、成交量比、跳空幅度、均线偏离、RSI 和星期几。

对应实现：backend/ml/features.py 和 backend/ml/features_v2.py
高频 4老师：你怎么避免前视偏差？

这是必须背熟的一题。 价格类技术指标大多使用 shift(1)，也就是第 N 天的特征只能来自 N-1 天及以前的数据； 标签则通过未来价格生成，例如 T+5 收盘价是否高于当前收盘价； 新闻特征先对齐交易日，再做聚合，确保不会把未来新闻泄露进来； 如果用了文本降维特征，TF-IDF 和 SVD 只在训练窗口拟合，测试窗口只做 transform，不会偷看测试期词汇分布。

对应实现：backend/ml/features.py、backend/ml/features_v2.py、backend/ml/walk_forward.py
高频 5老师：为什么你的标签设成 T+5？

因为 T+5 介于太短和太长之间。 T+1 容易受噪声和短期波动影响； T+10 又可能让新闻和价格之间的关系变弱； T+5 更适合作为短中期方向预测的折中选择，也更符合课程设计里“能解释、能回测”的要求。

相关标签：target_t1、target_t5、target_up_big_t5
高频 6老师：为什么还要做低噪声标签？

因为原始涨跌标签里，很多小幅波动本质上接近噪声。 如果把这些样本都当成正负样本，模型很难学出稳定映射。 所以我额外构造了涨幅阈值更高的标签，用来验证“是不是标签噪声拖累了排序能力”。

报告里对应 AUC 优先实验和多档收益阈值标签

5. 模型训练与验证，重点是你为什么这么选

这一部分老师通常会问“为什么不用别的模型”“参数怎么定的”“怎么验证不是过拟合”。 回答时不要空泛，要回到数据形态和验证方式。

高频 7老师：为什么选 XGBoost，而不是 LSTM 或 Transformer？

因为你的输入本质上是交易日粒度的结构化表格，不是大规模连续序列。 XGBoost 对中小规模表格数据更稳，训练快，可解释性也更好； 而 LSTM 对样本量、序列质量和调参更敏感，在当前数据条件下不一定占优。 我的拓展实验也验证了这一点，LSTM 并没有超过树模型。

对应训练：backend/ml/model.py、backend/ml/lstm_model.py
高频 8老师：你怎么做模型验证？

我不用随机切分，而是采用时间顺序和扩展窗口验证。 原因很简单：金融时间序列不能乱序，不然会把未来数据混进训练集。 扩展窗口的方式是先用历史较少的数据训练，再逐步扩大训练窗口，用更接近实战的方式观察泛化能力。

对应实现：backend/ml/walk_forward.py
高频 9老师：参数为什么是这些值？

这些参数并不是拍脑袋定的，而是结合扩展窗口搜索结果和课程项目的稳定性选择的。 比如 max_depth、n_estimators、learning_rate、subsample 和 colsample_bytree 都是为了在欠拟合和过拟合之间折中。 如果老师继续追问，你可以强调：我关注的是验证集表现和策略回测，而不是一味把训练分数调高。

常见参数：max_depth=4、n_estimators=200、learning_rate=0.05、subsample=0.8、colsample_bytree=0.8
高频 10老师：准确率不高，为什么还要做回测？

因为分类准确率和交易收益不是一回事。 分类指标衡量的是“猜对没猜对”，回测衡量的是“这个概率输出能不能形成可执行的交易决策”。 有时候模型整体准确率一般，但在高置信度样本上仍然有交易价值，所以必须同时看分类指标和策略指标。

评估指标：Accuracy、Precision、Recall、F1、ROC-AUC、年化收益、最大回撤

6. 回测与收益计算，老师很可能顺着问细节

回测是最容易“听起来会，细问就露馅”的部分。 你至少要能把仓位规则、手续费、收益和最大回撤讲清楚。

6.1 交易信号怎么来的？

模型输出上涨概率 prob_up。当它大于等于阈值时做多，低于阈值时空仓。也就是说，策略本质上是 long/cash，而不是复杂的多空对冲。

6.2 手续费怎么处理？

每次换仓都会扣手续费，课程里使用的是固定费率。这样做是为了更接近真实交易，不至于把无成本回测看得过于乐观。

6.3 为什么选择阈值而不是直接全仓？

因为模型并不是每天都足够有把握。通过阈值可以只在高置信度时入场，减少噪声交易，也更容易控制回撤。

6.4 最大回撤怎么理解？

它衡量的是净值从历史高点到后续低点的最大跌幅。这个指标比单纯收益更能体现策略的风险暴露，答辩时一定要提。

高频 11老师：你的回测到底怎么跑的？

你可以这样答： 先把模型输出的概率转换成交易信号； 然后按照日频 close-to-close 的简化假设，信号在下一交易段生效； 持仓期间按日收益累计，换仓时扣手续费； 最后得到净值曲线，再计算年化收益率、最大回撤、胜率和累计收益。

对应实现：backend/ml/strategy_backtest.py，重点看 simulate_equity_curve()
高频 12老师：为什么回测结果能达标？

我会诚实地说，这个结果是课程实验口径下的样本外回测结果，不等于实盘可直接复现。 它说明的是：在当前数据、标签和阈值设定下，这个策略组合在实验区间内达到了课程要求。 如果老师问稳定性，我会补一句：后续仍需要进一步验证跨时间段和跨股票的泛化能力。

课程结果文件：backend/ml/models/*_strategy_backtest.json

7. 前端展示和交互，答辩时也会被问

如果老师看见你的页面，很可能会问“这个图表后面接的是什么接口”“点击后发生了什么”。 你不需要背前端所有细节，但要知道主入口在哪里。

前端 1老师：前端上这些预测和分析信息从哪来？

主要来自后端 API。 比如预测结果从 /api/predict/{symbol}/forecast 来； 策略回测结果来自 /api/predict/{symbol}/strategy-backtest； 深度分析则通过单条新闻或区间分析接口获取，再在前端面板里展示。

对应前端：frontend/src/components/PredictionPanel.tsx、frontend/src/components/RangeAnalysisPanel.tsx、frontend/src/components/StrategyBacktestPanel.tsx

因为答辩里不仅要证明“模型能跑”，还要证明“系统能用”。 前端把新闻、预测、相似历史区间和深度分析集中到一个界面，能让用户更直观看到 AI 分析链路，而不是只看命令行输出。

相关组件：CandlestickChart.tsx、NewsPanel.tsx、SimilarDaysPanel.tsx

8. 高频追问与补刀题

这一段是你在老师追着问的时候用的。建议不要逐字背，但要把每个答案的逻辑记住。

追问 1老师：LLM 会不会胡说？

会，所以我没有让它直接做交易决策。 我让它做的是结构化信息提取和解释补充，真正的方向判断还是交给统计模型和回测结果。 这样做可以把幻觉风险限制在“解释层”，不会直接污染最终信号。

追问 2老师：为什么要做事件类别特征？

因为不同新闻类型对价格的影响机制不一样。 财报更直接影响盈利预期，监管新闻更容易带来风险溢价变化，产品发布更偏成长预期。 把这些类别单独拆出来，比简单统计正负新闻更有信息量。

追问 3老师：为什么还要做板块联动？

因为个股有时当天没新闻，但同板块龙头的消息会影响它。 尤其是科技和半导体板块，股票之间存在明显联动。 所以我把同板块其他股票的新闻情绪也作为辅助特征。

追问 4老师：你这个模型最大的局限是什么？

最大的局限是标签噪声和样本规模。 很多日频涨跌本身接近随机，短期价格波动很难完全被新闻解释； 所以这个系统更适合作为课程实验和研究原型，而不是直接宣称能稳定实盘盈利。

追问 5老师：如果让你继续优化，你会怎么做？

我会优先做三件事： 第一，继续减少标签噪声，优化阈值化标签； 第二，增强文本和事件特征； 第三，继续扩大跨股票、跨周期的扩展窗口验证，看看模型是否真有稳定泛化能力。

9. 60 秒总结稿，最后收口用

你可以直接背这段

“我的课程设计是一个基于 AI 算法的量化交易策略系统。系统先从 Polygon.io 获取股票行情和相关新闻，再通过规则过滤、LLM 情绪分析和交易日对齐，把新闻转成结构化特征。随后将这些新闻特征与价格技术指标一起输入 XGBoost，预测未来 T+1 或 T+5 的上涨概率，并将概率转换为交易信号进行样本外回测。为了保证结果可信，我重点控制了前视偏差、时间泄露和回测手续费问题，同时在前端提供了单条新闻深度分析和区间分析，让系统既能做预测，也能解释预测。总体来说，它不是简单的新闻聚合工具，而是一条完整的‘数据获取-分析-建模-回测-展示’链路。”
